[ Build-Tool ]
빌드 툴은 프로젝트를 빌드하는데 필요한 모든 작업을 자동화 하는 도구이며, 빌드 자동화말고도 그 외 프로젝트 작업도 자동으로 해준다.
Spring 에서는 빌드 툴을 Maven 과 Gradle 을 가장 많이 사용한다.

빌드 툴의 빌드 과정을 상세하게 나열하면 다음과 같다.
1. .java 소스 코드를 .class 파일로 컴파일
2. 테스트 실행
3. 라이브러리 다운로드
4. JAR/WAR 패키징
5. 실행
이와 같은 작업을 빌드 툴이 자동으로 수행해준다.
만약 빌드 툴이 없다면 수많은 라이브러리들을 직접 다운로드 하고, java 소스들을 class 파일로 컴파일하고 jar 로 묶은 다음 실행해주어야 한다.
프로젝트가 커질수록 라이브러리 관리와 빌드 과정이 많이 복잡해지는데, 빌드 툴은 이러한 과정을 설정 파일 하나로 자동화 해준다.

- 컴파일
자바 소스 코드를 바이트 코드로 컴파일 할 때 javac 를 활용할 수 있지만,
gradle build 나 mvn package 를 통해 한번에 컴파일 할 수 있다.

- 테스트 실행
테스트 파일 ( ~~Test.java ) 에 대해 gradle test , mvn test 를 수행하면,
JUnit 테스트를 모두 수행한다.

- 의존성 관리 및 라이브러리 다운로드
개발자는 필요한 라이브러리를 의존성 관리 파일에 기입하면, 빌드 툴은 해당 문서를 참고하여
필요한 라이브러리들을 저장소에서 다운로드한다.

여기서 저장소는 Maven Repository 등이 있다.

- JAR/WAR 패키징
gradle bootJar 혹은 mvn package 명령어를 수행하면, project.jar 혹은 project.war 로 패키징한다.

- 실행
gradle bootRun 혹은 mvn spring-boot:run 명령어를 통해 패키징된 어플리케이션을 실행할 수 있다.

[ 프로젝트 작업 자동화 ]
빌드 툴은 빌드 자동화 외에 이전 빌드 삭제 ( clean ) , 테스트 수행 ( test ) , 전체 빌드 ( build ) , 라이브러리 배포 ( publish ) ,
테스트 커버리지 생성 ( jacocoTestReport ) 등도 수행한다.

[ Maven 과 Gradle 의 차이 ]
Maven 과 Gradle은 모두 빌드 도구이지만, 설계 철학과 동작 방식이 다르다.
기존 Spring 환경에서는 Maven 을 많이 사용했지만, 시간이 지나 Gradle 을 사용하는 프로젝트가 많아지고 있다.

- Maven
Maven 은 대기업이나 SI 환경에서 기존 시스템과의 호환성 때문에 Maven 을 주로 사용한다.
Maven 은 정해진 방식대로 안정적이고 일관적인 빌드를 추구하고 있으며, 이에 대한 특징은 다음과 같다.

1. XML 기반
우선 Maven 은 XML을 사용한다. 구조가 정해져 있어 읽기 쉽지만 코드가 길어진다는 단점을 가지고 있다.

2. 선언형
Maven 은 선언형으로써 if문 혹은 for문과 같은 로직을 작성할 수 없다.
즉, 정해진 규칙 안에서만 동작할 수 있다.

3. 낮은 빌드 속도
Maven은 빌드를 할 때 마다 clean > Compile > Test > Package > Install 의 순서를 항상 거치게 된다.
그렇기 때문에 프로젝트의 규모가 커질수록 빌드의 속도가 느려진다.

만약 1000개의 Java 파일 중에서 1개만 수정되었다 하더라도, 전체를 지우고 다시 빌드한다.

4. 순차 빌드
만약 여러 모듈을 빌드를 할 경우 순차적으로 빌드를 수행한다.
즉, 병렬로 빌드를 수행할 수 없다.

5. 쉬운 학습 난이도
Maven은 dependency 와 plugin, repository 등 비교적 학습할 내용이 적지만, Gradle은 프로그래밍 언어이기 때문에
변수나 문법 등을 학습해야 하기에 초반에는 난이도가 있다.

- Gradle
프로젝트가 커질수록 빌드 속도와 설정의 유연성에서 큰 장점을 가지고 있다.
Gradle은 필요에 따라 빌드 과정을 프로그래밍으로 자유롭게 확장하는 철학을 가지고 있으며, 이에 대한 특징은 다음과 같다.

1. Groovy DSL 혹은 Kotlin DSL 기반
Gradle은 Groovy DSL 혹은 Kotlin DSL 을 사용한다. 코드가 짧고 간결하다는 장점을 가지고 있다.

2. 프로그래밍 가능
Gradle 은 빌드 스크립트 자체가 프로그래밍 언어이기 때문에 조건문이나 반복문을 사용할 수 있다.
그렇기 때문에 빌드 과정을 코드처럼 작성할 수 있다.

이를 활용하여 운영 환경과 개발 환경에 따라 다른 설정을 할 수 있어 유연한 설정이 가능하다.
# Profile에 따라 설정을 달리 할 수 있다. 근데 Maven 도 Profile 기반 설정이 가능하지만 표현력에 한계가 있따.

3. 빠른 빌드 속도
Gradle은 Incremental Build ( 증분 빌드 ) 를 지원하기 때문에 수정된 Java 파일만 컴파일 한다.
만약 1000개의 Java 파일 중에서 1개만 수정되었을 경우 수정된 Java 파일만 다시 컴파일하기 때문에 속도가 빠르다.

이는 전체를 clean 하고 다시 compile 하는 Maven 과는 다르다.

4. Build Cache
기존에 빌드한 결과물과 새로 빌드할 내용이 같다면, 다시 빌드를 수행하지 않고 기존 빌드한 결과물을 재활용 한다.

5. 병렬 빌드
만약 여러 모듈을 빌드할 경우 Gradle은 멀티 코어 CPU 를 활용하여 동시에 빌드를 수행한다.

6. Dependency Resolution
정해진 규칙대로 Dependency를 다운로드 하기만 하는 Maven 과는 다르게 버전이 충돌하지 않게 유연하게 적용한다.

7. 더욱 간단한 멀티 프로젝트 관리
Maven 도 멀티 프로젝트를 묶고 관리할 수 있지만 설정이 복잡하다는 문제를 가지고 있었다.
하지만 Gradle 은 include("project-name") 한줄로 프로젝트를 묶을 수 있다.
