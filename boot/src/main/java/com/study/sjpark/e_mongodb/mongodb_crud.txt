[ Database 명령어 ]
show dbs : 데이터베이스 목록 조회
db : 현재 사용하고 있는 데이터베이스
db.stats() : 현재 사용하고 있는 데이터베이스 메타데이터
use testdb : testdb 데이터베이스를 사용, 만약 해당 데이터베이스가 없을 경우 새로 생성
# use DATABASE_NAME 을 통해 생성된 데이터베이스는 최소 1개의 Document를 만들어야 show dbs 를 통해 볼 수 있다.
db.dropDatabase() : 현재 선택되어 있는 데이터베이스 삭제

[ Collection 명령어 ]
db.createCollection(name , [ Option ] ) : 새로운 컬렉션 생성
    - name : 컬렉션의 이름
    - Option : 컬렉션의 설정 값들이다. ( 필수는 아님 )
Option에 들어갈 수 있는 설정들
- capped ( boolean ) : true로 설정 시 고정된 크기의 컬렉션을 사용, 용량을 초과할 경우 가장 오래된 데이터를 덮어씀
- size ( number ) : 해당 컬렉션의 최대 사이즈 ( byte 단위이며 capped 가 true 일 경우 설정 필요 )
- max ( number ) : 해당 컬렉션에 저장될 수 있는 최대 Document 갯수

ex ) db.createCollection('member' , {capped : true , size : 6142800, max:10000})

show collections : 생성된 모든 컬렉션들을 조회
db.COLLECTION_NAME.drop() : 선택된 데이터베이스의 해당 컬렉션을 제거

[ Document 명령어 ]

- 데이터 삽입 ( Create )
db.COLLECTION_NAME.insert() : 1개 이상의 Document를 삽입하는 명령어, 만약 Collection 이 없다면 자동으로 생성 후 Insert 한다.
ex ) db.member.insert({'name' : 'sjpark' , 'age' : 26})
ex ) 여러 개를 넣을 때는 배열을 쓴다. db.member.insert([{'name' : 'sjpark' , 'age' : 26} , {'name' : 'hbjang' , 'age' : 25}])

db.COLLECTION_NAME.insertOne() : 1개의 Document를 삽입, 만약 Collection 이 없다면 자동으로 생성 후 Insert 한다.
db.COLLECTION_NAME.insertMany() : 2개 이상의 Document를 삽입, 만약 Collection 이 없다면 자동으로 생성 후 Insert 한다.

- 테이터 조회 ( Read )
db.COLLECTION_NAME.find() : Document 리스트를 출력
db.COLLECTION_NAME.find().pretty() : Document 리스트를 이쁘게 출력
db.COLLECTION_NAME.find({"name" : "A"}) : Document 중에서 name 의 값이 A 인 Document를 조회
db.COLLECTION_NAME.findOne() : Document의 조회 결과 중 첫번째 데이터만 가져온다.

>> 논리 연산자
{ 필드: { $gt: 값 } } // 필드 > 값
{ 필드: { $lt: 값 } } // 필드 < 값
{ 필드: { $gte: 값 } } // 필드 >= 값
{ 필드: { $lte: 값 } } // 필드 <= 값
{ 필드: { $eq: 값 } } // 필드 == 값
{ 필드: { $ne: 값 } } // 필드 != 값
{ 필드: { $in: [ 값1, 값2, 값3, ... ] } // 필드 == (값1 or 값2 or 값3)
{ 필드: { $nin: [ 값1, 값2, 값3, ... ] } // 필드 != (값1 and 값2 and 값3)
ex ) db.member.find({"name" : { $in: ['sjpark' , 'hbjang'] } , 'age' : { $gte : 25 }}) # 이름은 sjpark 이거나 hbjang 이여야 하고, 나이는 25 이상이여야 한다.
{ $or: [ { 조건1 }, { 조건2 }, ... ] } // or 조건
ex ) db.member.find({ $or : [ {"name" : "sjpark"} , {"name" : "hbjang"} ] })
{ $and: [ // and 조건
  { $or: [{ 조건1 }, { 조건2 }] },
  { $or: [{ 조건3 }, { 조건4 }] }
] }
{ $nor: [{ 조건1 }, { 조건2 }, ...] } // not or ( 모든 조건이 만족하지 않음 )
{ $not: { 조건 } } // 조건이 아닌값.  $nor의 단일 버전

>> Where 조건
자바스크립트 문법을 활용하여 검색할 수 있다. 단, this 연산자를 무조건 사용해야 한다.

아래의 두 명령어는 같은 결과 값을 반환한다.
db.member.find({ $or : [ {"name" : "sjpark"} , {"name" : "hbjang"} ] })
db.member.find({ $where : "this.name == 'sjpark' || this.name == 'hbjang'"})

>> 정렬
db.member.find().sort({"age" : -1})
age 필드를 기준으로 정렬, 1은 오름차순, -1는 내림차순

>> 선택 필드 Projection
조회 결과 중 특정 필드만 가져오도록 필터링할 수 있다.
find() , findOne() 명령의 두번째 인자에 어떤 필드만 가져올 건지 입력하면 된다.
기본 값은 true 이며, false로 설정 시 해당 필드는 가져오지 않는다.

ex ) db.member.find({} , {"_id" : false , "name" : false })

- 데이터 수정 ( Update )

db.COLLECTION_NAME.update({ 조건 } , { $set: { 수정할  필드.. } } , { 옵션.. })
ex ) db.member.update({"age" : 25} , { $set: { "age" : 99 , "metadata" : "data" } })
위의 내용은 age 가 25인 Document에 대해 age 를 99로 변경하고 metadata 라는 key를 추가하는 명령이다.
기본적으로 조건에서 조회되는 첫번 째 Document만 수정이 된다. 즉, 조건에 맞는 Document 가 여러 개 일 경우에도 가장 첫번째 Document 만 수정된다.

옵션에 들어갈 수 있는 설정 값은 다음과 같다.
upsert : 조건에 맞는 데이터가 없을 경우 insert 를 할 것인지에 대한 여부
ex ) db.member.update({"age" : 1234} , { $set: { "age" : 12 , "metadata" : "data..." } } , { upsert : true })


- 데이터 삭제 ( Delete )

db.COLLECTION_NAME.remove() : 모든 Document를 삭제
db.COLLECTION_NAME.remove({ 조건 }) : 조건에 맞는 Document를 삭제
ex ) db.member.remove({"age" : { $gt : 30 }})

db.COLLECTION_NAME.deleteOne() : 조건에 맞는 Document 중 첫번 째 Document 만 삭제
db.COLLECTION_NAME.deleteMany() : 조건에 맞는 모든 Document 삭제

# remove() 명령은 기본적으로 다중 삭제이기 때문에, 보다 더 안전한 deleteOne() 과 deleteMany() 를 사용하는 것이 바람직 하다.
