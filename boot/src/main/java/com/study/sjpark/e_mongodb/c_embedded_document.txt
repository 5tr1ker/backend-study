[ Embedded Document 란 ]
Embedded Document는 Document 안에서 Field가 배열 타입인 Document를 말한다.
아래의 Document에서는 car 이 Embedded Document 가 된다.
{
  _id: ObjectId('6a5e133091612abd0a62d435'),
  age: 42,
  isDelete: false,
  car: [
    {
      bmw: 'M3'
    },
    {
      Hyundai: 'Avante'
    },
    {
      KIA: 'Sorrento'
    }
  ],
  language: [
      'eng',
      'kor',
      'jpn'
  ],
  metadata: {
    country: 'kor'
  }
}

[ 요소 내 검색 ]
- $elemMatch : 특정 Embedded Document 중에서 조건이 배열 안의 요소와 일치하는 Document를 검색
db.COLLECTION_NAME.find({car : { $elemMatch : {'bmw' : 'M3'} }})
# 만약 일반 배열인 경우 db.COLLECTION_NAME.find({'language' : 'eng'}) 으로 검색할 수 있다.

- $all : 해당 요소를 모두 가지고 있는 배열을 가진 Document를 검색
db.COLLECTION_NAME.find({language : { $all : ['eng' , 'kor'] }})
# 위의 예시에서 language 필드의 배열 요소로 'eng' 과 'kor' 를 모두 가지고 있는 Document를 검색한다.

- $size : 배열의 크기가 조건과 일치하는 Document를 검색
db.COLLECTION_NAME.find({language : {$size : 3}})

# 위의 예시에서 language 필드의 배열 요소 갯수가 3인 Document를 검색한다.

# 만약 2개 이상의 depth 로 들어가 검색을 하고 싶을 때 .(점) 을 이용해서 특정 필드를 선택한다.
# ex ) db.member.find({'metadata.country' : 'kor'})
