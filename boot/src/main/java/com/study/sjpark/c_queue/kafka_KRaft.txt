[ KRaft 모드 ]
KRaft 모드 ( Kafka Raft 모드 ) 는 Zookeeper 없이도 Kafka 내부에서 클러스터를 운영하고 메타데이터를 관리하는 방식이다.
기존에는 Kafka 클러스터를 운영하기 위해서는 ZooKeeper가 필요했었다. ZooKeeper는 브로커 등록, 컨트롤러 선출, 메타데이터 저장 ( Topic, Partition 등 )
역할을 하였지만 서버가 여러대 있어 운영이 복잡하고 장애 포인트가 많았으며, 확장성과 성능에 한계가 있었다.

이때 KRaft 모드를 활용하면 Zookeeper가 없어도 Kafka 내부에서 Raft 기반 메타데이터 관리 시스템을 활용하여 관리한다.

[ KRaft 구조 ]
KRaft에서는 브로커 중 일부가 Controller 역할을 수행한다.
KRaft의 구성 요소로는 크게 Controller Quorum 와 Broker로 나뉜다.

Controller Quorum은 메타데이터를 관리하고 Raft로 리더를 선출하고, Broker는 실제 메세지를 저장하고 처리한다.
이렇게 함으로써 운영이 단순화되고, 장애 복구 속도가 향상되었으며, 확장성도 좋아졌다. 그리고 더 빠른 메타데이터 처리가 가능하게 되었다.
